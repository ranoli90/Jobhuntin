This document summarizes **all errors reported** by the full-stack audit (TypeScript, ESLint, depcheck, knip, npm audit, Ruff, Bandit, pip-audit). Source: `audit-output.txt` (generated by `npm run audit:full:skip-install`).

## 1. Frontend — TypeScript (`tsc --noEmit`)

### 1.1 File name casing (TS1261, TS1149) — **Critical for Linux/CI**

**Issue:** The same file is imported with different casing. On Windows this works; on Linux and in strict TypeScript it fails.

| Correct file (on disk) | Wrong import (lowercase) | Files using wrong import |
|------------------------|---------------------------|---------------------------|
| `Button.tsx` | `@/components/ui/button` | agent-improvements/*, communications/*, user-experience/*, app pages (agent-improvements, dlq-dashboard, screenshot-capture, communication-preferences, notification-history, pipeline-view, application-export, follow-up-reminders, interview-practice, multi-resume, application-notes, communications, email) |
| `Card.tsx` | `@/components/ui/card` | Same set as above |
| `Badge.tsx` | `@/components/ui/badge` | Same set as above |
| `Input.tsx` | `@/components/ui/input` | OAuthHandler, ScreenshotCapture, AlertProcessor, BatchProcessor, ApplicationExport, PipelineView, email |

**Fix:** Use consistent PascalCase in all imports, e.g. `@/components/ui/Button`, `Card`, `Badge`, `Input`. Do **not** use `button`, `card`, `badge`, `input` in path.

## 4. Frontend — Knip (dead code)

**Status:** Knip **crashed**.

**Error:**  
`ReferenceError: __dirname is not defined in ES module scope` in `vite.config.js` when Knip (ESM) loads it.

**Fix:** Either rename `vite.config.js` to `vite.config.cjs` so it’s loaded as CommonJS, or in `vite.config.js` replace `__dirname` with ESM equivalent, e.g.:

```js
import { fileURLToPath } from 'url';
import { dirname } from 'path';
const __dirname = dirname(fileURLToPath(import.meta.url));
```

## 7. Backend — Bandit (security)

**Summary:** **9 High, 68 Medium, 80 Low** (by severity); **89 High confidence** issues.

### High (examples)
- **B324 (hashlib):** Use of MD5 for security — in `shared/db_load_balancer.py`, `shared/db_query_monitor.py`, `shared/db_retry_handler.py`.  
  **Fix:** Use `hashlib.md5(..., usedforsecurity=False)` for non-security hashing, or switch to SHA-256 for security-sensitive use.

### Medium (examples)
- **B307 (eval):** `eval(plan)` in `shared/db_query_monitor.py` — use `json.loads` or `ast.literal_eval` instead.
- **B108:** Hardcoded `/tmp/db_cache` in `shared/db_query_cache.py` — use env or config.
- **B608:** Possible SQL injection via string-built queries (e.g. `shared/monitoring_service.py`, `shared/pagination.py`, `shared/redis_cache.py`, `shared/memory_cache.py`) — use parameterized queries.
- **B301/B403:** `pickle` in `shared/memory_cache.py`, `shared/redis_cache.py` — avoid unpickling untrusted data; consider JSON or other format.
- **B607/B603:** `subprocess` / partial path in `shared/virus_scanner.py` — use full path and validate input.

### Low (examples)
- **B311:** `random` for non-crypto use (e.g. load balancing, jitter) — acceptable for non-security; document or use `secrets` if ever used for tokens.
- **B110:** Try/except/pass — in `shared/event_tracking.py`, `shared/metrics.py`; add logging or re-raise where appropriate.
- **B404:** `subprocess` import — acceptable when used safely (e.g. virus_scanner).

**Fix:** Address High first (MD5, eval); then parameterize SQL and replace pickle in security-sensitive paths; then hardcoded paths and try/except/pass.

## Fixes applied (post-audit)

- **Import casing:** All `@/components/ui/*` imports updated to PascalCase (Button, Card, Badge, Input, Alert, Label, Textarea, Select, Progress, Tabs, Checkbox, Switch). Added `Switch.tsx` and removed duplicate `switch.tsx`.
- **markdownify:** `requirements.txt` updated to `markdownify>=0.14.1` (CVE-2025-46656).
- **Python Bandit (MD5):** All `hashlib.md5(...)` calls updated to use `usedforsecurity=False` in `shared/api_response_cache.py`, `shared/db_query_monitor.py`, `shared/db_load_balancer.py`, `shared/db_retry_handler.py`, `packages/backend/domain/cache_manager.py`.
- **Python eval:** `shared/db_query_monitor.py` — replaced `eval(plan)` with `json.loads(plan)` and added `import json`.
- **api_response_cache.py:** Fixed `.cache` typo (use `.hexdigest()` for cache filenames); removed duplicate `get_response_cache` / `init_response_cache` block.
- **application_status_fix.py:** Added full imports and stubs so the fragment parses (router, TenantContext, _get_tenant_ctx, _get_pool, logger).
- **vite.config.ts:** Added ESM-safe `__dirname` via `fileURLToPath(import.meta.url)` for Knip/ESM.
- **BatchProcessor.tsx:** Removed duplicate `average_processing_time` in interface and in `mockStats` object.
- **AlertProcessor.tsx:** Added `Switch` import; typed `onCheckedChange` callback parameter as `(checked: boolean)`.
- **ScreenshotCapture.tsx:** `onCheckedChange={setFullPage}` → `onCheckedChange={(checked: boolean) => setFullPage(checked)}`; added types for `e` in onChange/onKeyPress.
- **TypeScript implicit any:** Added `React.ChangeEvent<HTMLInputElement>`, `React.MouseEvent<HTMLButtonElement>`, `React.KeyboardEvent<HTMLInputElement>` where needed in application-export, OAuthHandler, AlertProcessor, ApplicationsView.
- **Index signatures (TS7053):** Added `as keyof typeof map` in DocumentProcessor, OAuthHandler, AlertProcessor, PerformanceMetrics, BatchProcessor for string-indexed lookups.
- **ESLint:** `.eslintrc.json` extends updated to `plugin:@typescript-eslint/recommended` and `plugin:@typescript-eslint/recommended-requiring-type-checking`.
